# 1.0.0 (2023-03-23)

这是 crane4j 的第一个正式版本，如果遇到问题可以在群内或 issues 中反馈，作者会尽快响应。

具体内容参见：Milestone (opens new window)。

Feature

  • 提供基于 Guava 的 LoadingCache 的缓存支持 (opens new window)
  • 添加默认的组合注解扩展包 (opens new window)；
  • 支持使用Spring的@Order注解对装配操作排序 (opens new window)；
  • 支持在注解中通过 beanName 引用 Spring 上下文中的组件 (opens new window)；
  • 添加扩展模块，支持基于 MybatisPlus 自动生成表查询数据源 (opens new window)；

Refactor

  • 重构装配处理器，并统一为所有装配操作提供一对一、一对多、多对多映射支持 (opens new window)；
  • 重构并完善缓存功能 (opens new window)；
  • 添加容器工厂组件以隔离和丰富获取数据源容器的渠道 (opens new window)；
  • 将注解分离至独立的 crane4j-annotation 模块 (opens new window)；

Test

  • 补充 MultiKeyAssembleOperationHandler 的测试用例 (opens new window);
  • 添加示例项目 (opens new window)；

docs

  • 添加官方站点配置 (opens new window)；
  • 代码注释国际化 (opens new window)；
  • 提供 crane4j 的官方文档站点 (opens new window)；

chore

  • 重命名 groupId 为 cn.crane4j (opens new window)；
  • 添加自动化 CI 流程 (opens new window)；

# 1.1.0 (2023-03-30)

这一个重构与增强版本，增强了一些新功能，并且调整了一部分 api 的使用方式。

具体内容参见：Milestone (opens new window)。

Feature

  • core模块应该默认支持ognl表达式 (opens new window)；
  • 简化@Mapping配置，可以在一个属性同时配置src和ref (opens new window)；
  • 字段映射支持以链式操作符获取或设置多级嵌套对象的属性 (opens new window)；
  • 提供基于 Guava 的 LoadingCache 的缓存支持 (opens new window)；
  • 支持通过 @ContainerConstant 注解的配置，反转基于常量类构建的容器键值 (opens new window)

Refactor

  • 重构并完善缓存功能 (opens new window)；
  • 将starter模块中的部分组件功能分离为核心模块中的独立组件 (opens new window)

docs

  • 文档重构 (opens new window)；

# 1.2.0 (2023-04-09)

这一个重构版本，增强了一些新功能，并且调整了 MybatisPlus 扩展的引入方式，如果已经使用了 MybatisPlus 插件，则需要按文档重新引入。

具体内容参见：Milestone (opens new window)。

Feature

  • 支持直接填充 Map 类型的数据 (opens new window)；
  • 提供 @AssembleMp 注解，为基于 MP 的数据源提供更好的支持 (opens new window)；
  • 允许自定义注解，支持解析自定义的配置规则 (opens new window)；
  • MpBaseMapperContainerRegister支持懒加载 (opens new window)；

Refactor

  • 将 @Assemble 和 @Disassemble 的解析逻辑分离到独立的注解处理器 (opens new window)；
  • 重构项目结构，提供不同环境下的最小依赖 (opens new window)；

Fix

  • 当Bean被spring代理时，调用方法数据源容器报错 (opens new window)；

# 1.3.0-ALPHA (2023-05-10)

这 1.3.0 的预览版本，重构和增强了一些已有功能，并添加了一些新的功能。

其中，基于新特性添加基于接口代理的填充方法 (opens new window)，crane4j 将更好的支持处理 JSONObject 或 Map 类型的非 JavaBean 对象。

具体内容参见：Milestone (opens new window)。

Feature

  • 提供 @AssembleEnum 注解，对枚举类型数据源提供更好的支持 (opens new window)；
  • 提供一个不基于 ThreadLocal 的动态数据源容器提供者 (opens new window)；
  • 代理填充方法支持设置临时容器 (opens new window)；
  • OperatorProxyFactory中代理方法的生成也应当支持多种策略 (opens new window)；
  • 支持通过 Spring 依赖注入获取被 @Operator 注解的接口的代理对象 (opens new window)；
  • 装配操作中的容器支持懒加载 (opens new window)；
  • 添加基于接口代理的填充方法 (opens new window)；
  • 提供一个默认的可配置容器注册者实现 (opens new window)；

Refactor

  • 减少对 Hutool 的依赖 (opens new window)；
  • 移除 OperationAnnotationResolver 级别的配置缓存 (opens new window)；
  • 配置解析器应当支持所有的AnnotatedElement类型对象的注解配置 (opens new window)；

Test

  • 提高测试覆盖率 (opens new window)；
  • MybatisPlus相关扩展的测试用例数据库更换为H2 (opens new window)；
  • 修复了引入 crane4j-spring-boot-starter 时的一些自动装配问题；

Doc

  • 补充文档，如何在非 spring 环境和 spring 环境使用 crane4j (opens new window)；
