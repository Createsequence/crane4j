## 对象内省

默认情况下，若我们在 `@Assemble` 中未指定容器的 `namespace`，则解析时默认会指派一个 `EmptyContainer` 用于占位，不过实际执行时并不会调用容器，而是会进行对象内省，即**以待处理对象本身作为数据源**。比如说：

~~~java
public class Foo {
    @Assemble(props = @Mapping(ref = "alias"))
    private String name;
    private String alias;
}
~~~

上述操作将会将当前对象的 `name` 字段映射至 `alias` 上。

该功能一般用于同步冗余的别名字段。
